# SNI Checker для VLESS Reality

Утилита подбирает **лучший `dest`/SNI** для маскировки VLESS Reality на Вашей ноде.
Она проверяет домены-кандидаты и сразу показывает, какой из них поставить в конфиг,
чтобы соединение было стабильным и не «отваливалось».

Скрипт написан на чистом Python 3 (только стандартная библиотека) — **ничего
устанавливать не нужно**, он работает на любом сервере, где есть `python3`.

---

## Зачем это нужно

В Reality параметр `dest` — это «настоящий» сайт, под который маскируется Ваш сервер.
Если выбрать неудачный домен (нестабильный CDN, заблокированный или чужой прокси),
сервер начинает работать с перебоями: то подключается, то нет. Эта утилита находит
домен, который:

- поддерживает **TLS 1.3 + HTTP/2 + X25519** (обязательные требования Reality);
- имеет **валидный сертификат**;
- **не спрятан за Cloudflare/Akamai** (они дают нестабильный «отпечаток»);
- имеет **минимальную задержку** именно с Вашего сервера.

---

## Требования

- Сервер на Linux, где запущен Xray (запускать нужно **на самой ноде**, не на домашнем ПК — задержка измеряется с сервера).
- `python3` (есть почти везде по умолчанию).
- Желательно `openssl` и `curl` (для определения кривой X25519 и скачивания; обычно уже стоят).

---

## Установка и запуск за одну команду

Подключитесь к серверу по SSH и выполните:

```bash
curl -fsSL https://raw.githubusercontent.com/NikitaAzmov/SNICHECKER/main/sni-check.py -o /tmp/sni.py && python3 /tmp/sni.py
```

Команда скачает скрипт во временный файл и сразу его запустит. Будет проверен
встроенный список из ~100 доменов-кандидатов (мировые бренды + японские +
российские), а на экран выведутся **топ-15** лучших по задержке.

## Как пользоваться

### 1. Обычный запуск (рекомендуется)

```bash
python3 /tmp/sni.py
```

Проверяет встроенный список и выдаёт лучший вариант.

### 2. Проверить локальные домены страны Вашей ноды (рекомендуется!)

Для ноды в конкретной стране стоит проверить **локальные домены этой страны** —
у них минимальный пинг с сервера. Глобальные бренды проверяются всегда, а флаг
`--country` добавляет местные:

```bash
python3 /tmp/sni.py --country de            # Германия
python3 /tmp/sni.py --country fi us fr      # несколько стран сразу
python3 /tmp/sni.py --country eu            # вся Европа
python3 /tmp/sni.py --country all           # все страны (большой скан)
```

Посмотреть список доступных кодов стран:

```bash
python3 /tmp/sni.py --list-countries
```

Доступно 23 страны: 🇩🇪 de, 🇫🇮 fi, 🇺🇸 us, 🇫🇷 fr, 🇳🇱 nl, 🇸🇪 se, 🇬🇧 gb,
🇵🇱 pl, 🇮🇹 it, 🇪🇸 es, 🇳🇴 no, 🇩🇰 dk, 🇨🇭 ch, 🇦🇹 at, 🇨🇿 cz, 🇧🇪 be,
🇮🇪 ie, 🇵🇹 pt, 🇷🇴 ro, 🇹🇷 tr, 🇨🇦 ca, 🇯🇵 jp, 🇷🇺 ru.

### 3. Проверить свои домены

```bash
python3 /tmp/sni.py images.apple.com max.ru userapi.com
```

### 4. Проверить домены, найденные сканером RealiTLScanner

Если Вы предварительно просканировали подсеть своей ноды утилитой
[RealiTLScanner](https://github.com/XTLS/RealiTLScanner) и получили файл
`reality_sni.csv`, скормите его напрямую — скрипт сам отфильтрует мусор и
проверит каждый домен по-настоящему:

```bash
python3 /tmp/sni.py --csv /tmp/reality_sni.csv
```

### Дополнительные параметры

| Параметр | Назначение | По умолчанию |
|----------|------------|--------------|
| `--country КОДЫ` | Добавить локальные домены стран (`de fi us`, `eu`, `all`) | — |
| `--list-countries` | Показать доступные коды стран и выйти | — |
| `--top` | Сколько лучших доменов показать (`0` = все) | `15` |
| `--port` | Порт для проверки | `443` |
| `--timeout` | Таймаут на домен, сек | `6` |
| `--csv ФАЙЛ` | Загрузить домены из вывода RealiTLScanner | — |

Примеры:

```bash
python3 /tmp/sni.py --top 30      # показать топ-30 вместо 15
python3 /tmp/sni.py --top 0       # показать вообще все проверенные домены
```

## Что Вы получаете

Скрипт выводит таблицу, отсортированную по задержке, и в конце — **однозначный
лучший выбор** с готовыми строками для конфига:

```
DOMAIN                    PING     TLS1.3 H2  CURVE           CERT CDN        VERDICT
────────────────────────────────────────────────────────────────────────────────────
fallback.cdn-tinkoff.ru   9ms      OK     OK  X25519          OK   -          ✓ подходит
images.apple.com          18ms     OK     OK  X25519          OK   -          ✓ подходит
www.cloudflare.com        12ms     OK     OK  X25519          OK   Cloudflare ⚠ Cloudflare
some-vpn.example          —        NO     NO  ?               NO   -          ✗ не годится

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
🏆 ЛУЧШИЙ ВЫБОР: fallback.cdn-tinkoff.ru  (9ms, X25519)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Вставьте в realitySettings:
  "dest": "fallback.cdn-tinkoff.ru:443",
  "serverNames": ["fallback.cdn-tinkoff.ru"],
```

### Расшифровка колонок

| Колонка | Что означает |
|---------|--------------|
| **DOMAIN** | Проверяемый домен |
| **PING** | Время TCP-коннекта с Вашего сервера (меньше — лучше) |
| **TLS1.3** | Поддержка TLS 1.3 (обязательно `OK`) |
| **H2** | Поддержка HTTP/2 через ALPN (обязательно `OK`) |
| **CURVE** | Группа обмена ключами; для Reality нужен **X25519** |
| **CERT** | Валидность TLS-сертификата (обязательно `OK`) |
| **CDN** | Признак Cloudflare/Akamai (желательно `-`) |
| **VERDICT** | Итог: `✓ подходит` / `⚠ с оговоркой` / `✗ не годится` |

**Брать нужно строку `✓ подходит` с наименьшим PING.** Скрипт сам выберет её и
покажет в блоке «🏆 ЛУЧШИЙ ВЫБОР».

---

## ⚠️ ВАЖНО — прочитайте перед выбором ⚠️

> **❗ Зелёный `✓ подходит` — это НЕ гарантия 100% рабочего SNI ❗**
>
> Скрипт проверяет только технические требования Reality (TLS 1.3, HTTP/2,
> X25519, валидный серт, задержка). Он **не может знать**, насколько домен
> «живучий» в долгой перспективе и как именно его видит DPI Вашего провайдера.
>
> **Выбирайте из списка с умом:**
> - 🟢 Отдавайте предпочтение **крупным, известным брендам** (Apple, Yahoo,
>   Google, банки/CDN) — они стабильны и почти не блокируются.
> - 🔴 **Избегайте** доменов, похожих на чужие VPN/прокси (`*.vpn`, `*.tech`,
>   `*.fun`, случайные поддомены, динамические DNS вроде `*.mooo.com`,
>   `*.nip.io`) — даже если они `✓ подходит`, они нестабильны и сами уже
>   могут быть под подозрением.
> - 🔴 **Не берите** Cloudflare/Akamai (помечены `⚠`).
>
> **Обязательно тестируйте на практике:**
> 1. Поставьте выбранный домен в конфиг.
> 2. Проверьте реальное подключение клиентом несколько часов / в разное время суток.
> 3. Если соединение нестабильно — **вернитесь к списку и возьмите другой**
>    `✓ подходит` домен. Иногда нужно перебрать 2–3 варианта.
> 4. Идеально — держите **разные SNI на разных нодах**, чтобы не было единой
>    точки отказа.
>
> Этот инструмент сильно сужает выбор и отсекает заведомо плохие варианты,
> но финальное решение и проверка боем — за Вами.

---

## Как применить результат

1. Скопируйте строки `dest` и `serverNames` из блока «ЛУЧШИЙ ВЫБОР» в секцию
   `realitySettings` конфига Xray на ноде.
2. **Важно:** в панели **remnawave** в настройках Host для этого инбаунда
   поле **SNI** тоже поменяйте на этот же домен — иначе клиенты не совпадут с нодой.
3. Перезапустите ноду / примените конфиг.

## Частые вопросы

**Запускать на сервере или дома?**
Только на сервере (ноде) — задержка измеряется именно оттуда, и лучший домен на
разных серверах может отличаться.

**Колонка CURVE показывает `?`**
Значит на сервере нет `openssl` или он старой версии. Остальные проверки при этом
работают. Поставьте свежий `openssl` (`apt install -y openssl`), если нужна эта колонка.

**Все домены вышли `✗ не годится`**
Скорее всего на сервере проблемы с сетью/исходящими соединениями на 443 порт, либо
слишком маленький `--timeout`. Увеличьте таймаут: `python3 /tmp/sni.py --timeout 10`.

**Безопасно ли запускать через `curl | python3`?**
Это Ваш собственный репозиторий, код которого Вы видели — да, безопасно. Чужие
ссылки так запускать не стоит без просмотра кода.
